# instr_hints.txt -- one-line help per INSTR field, shown (inverted, ">>"-prefixed) on
# the bottom row of the instrument editor as the cursor moves. Edit freely; `make`
# validates it and regenerates build/instr_hints.i. Toggle on-console via OPTIONS -> HINTS.
#
# Rules: UPPERCASE only (no lowercase glyphs); <= 35 chars/line (the ">> " prefix is added
# by the renderer); no double-quotes; a too-wide line fails the build. A blank line = that
# field has no hint (the row goes blank). Lines within a section are in on-screen field
# order (top to bottom, then the FM operator-grid columns left to right).
#
# Sections map to the field layout in src/main.asm:
#   [COMMON] rows 0-1 for every type   : INST selector, TYPE
#   [FM]     voice_lbl (rows 2+)        : HLD VOL PAN TSP TBL TBS ALGO FB AMS FMS SWEEP
#   [FM.OP]  op grid columns (i_op)     : MUL DT TL RS AR AM D1 D2 RR SL
#   [TONE]   psg_off (rows 2+)          : VOL ATK HLD DCY TSP SWP VIB TRM TBL TBS MODE RATE
#   [NOISE]  psg_off (rows 2+)          : same set as TONE
#   [WAVE]   wave_off (rows 2+)         : WAVE VOL ATK HLD DCY WARP DRIVE FOLD CRUSH TBL TBS
#   [KIT]    (rows 2+)                  : KIT RATE TSP
#   [PERC]   (rows 2+)                  : BASE + per-operator frequency
